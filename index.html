    Step 1) Information exchange required before using the plugin

    • Information you need to provide to Identitymind To provide a secure implementation of the plugin IDM team needs to first set up a couple of attributes in the backend, for that purpose when we create a key we need to know:
    Attribute Description
    Usage Tier The type of service you want to purchase. The are two choices: Basic or 2ID
    Host where the plugin will be loaded The URL to the webpage to display the hosted ICO Plugin form. This has to be a publicly accessible URL or localhost. For example: https://amazingcrypto.com or http://localhost
    • Information Identitymind needs to provide to you
    Attribute Description
    x-api-key Alphanumeric string to identify yourself to the Identitymind system.
    To prevent clickjacking attacks you must set X-Frame-Options header to SAMEORIGIN or DENY where the plugin will load.

    Step 6) Configure the plugin

    Custom settings

    Besides setting the plugin_token you can also override and set different configurations.

    Custom configs example

    <script>
    var _idm = {
        container_id: "idm-container",
        plugin_token: "yyyyyy",
        on_response: function(jwtresponse){
            //handle response jwtresponse here
        },
        form_labels:{
            "first_name":"Custom First Name label",
            "email": "Custom email Label"
        },
        accept_message:"You were accepted, wohoo",
        deny_message:"ops, there was an error, please contact us.<a href=”mailto:c@ex.com">Contact Email</a>,
        existing_kyc_message: "You have already registered."

    };
    (function() {
        function ls() {
            var e=document.createElement('script');
            e.src='https://d1rs4puinhb994.cloudfront.net/idm.min.js';
            document.body.appendChild(e)
        }
        if(window.attachEvent){
            window.attachEvent('onload',ls)
        }else{
            window.addEventListener('load',ls,!1)
        }
    })()
</script>
    Name Type Description Default
    container_id string The container id on which the plugin will render. The default is idm-container. empty
    plugin_token string The token generated by AWS with the key provided by Identitymind empty
    form_labels object A list of fields ids and the label to be set empty
    styles object A list of fields ids and the css to be applied (inline format) empty
    accept_message string The message to show the user when the result is ACCEPT or MANUAL_REVIEW Registration successful. Thank you.
    deny_message string The message to show the user when the result is DENY Sorry we can’t create an account at this time please contact client support.
    existing_kyc_message string The message to show the user when there’s a KYC already in place. It seems that you have already signed up. If you think this is an error please contact client support.
    agree_checkbox string Text to show on the agree checkbox. It also accepts HTML tags. I have read and agree COMPANY_NAME <a href="">Terms and conditions</a>
    required_id_country_list (for 2ID tier only) list List of 2 digit country codes to ask for national ID and the regex pattern to verify the national id format of that country, separated by :: empty
    required_address_country_list list List of 2 digit country codes that when selected will show the full address form empty
    country_blacklist  list List of 2 digit country codes that will be removed from the country select empty
    country_whitelist list Will only load this list of 2 digit country codes in the country select  empty
    on_response function Function to handle response data (in JWT format). function(jwtresponse){}

    Example of required_id_country_list: ['US::^(?!666|000|9\\d{2})\\d{3}[- ]{0,1}(?!00)\\d{2}[- ]{0,1}(?!0{4})\\d{4}$']

    Note: you need to escape character like backslashes
